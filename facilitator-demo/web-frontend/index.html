üîÑ Complete X402 Workflow Demonstration

This demonstrates the complete end-to-end X402 payment flow:

1

Resource Discovery

Client discovers available paid resources from the resource server catalog.

2

Initial Request (402 Response)

Client attempts to access protected resource and receives 402 Payment Required.

3

Payment Payload Creation

Client creates payment payload with blockchain transaction details.

4

Request with Payment

Client retries request with X-Payment header containing payment payload.

5

Payment Verification

Resource server forwards payment to facilitator for verification.

6

Payment Settlement

Facilitator settles the payment on the blockchain network.

7

Resource Delivery

Resource server delivers protected content to the client.
